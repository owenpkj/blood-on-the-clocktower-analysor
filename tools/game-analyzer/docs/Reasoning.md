# 推理方法论（game-analyzer 工具侧）

本文档是对局分析工具的推理设计说明，不属于游戏规则。
游戏规则以 `knowledge/core_rules.md` 为准。

## 推理的数学本质

**约束满足问题：**

给定所有玩家的信息声称集合，求解所有可能的"世界状态"（身份分配），使得：

1. 身份分配符合角色配置规则
2. 不可靠信息源数量 ≤ K（上界）
3. 剩余信息在该状态下逻辑自洽

**不可靠信息源上界 K 的计算：**

```
K = 邪恶玩家数量 + 可能的醉酒位数量 + 可能的中毒位数量
```

## 推理阶段特征

| 阶段 | 特征 | 推理价值 |
|------|------|----------|
| 前期（1-2轮） | 信息稀少，善良可能伪装 | 低 - 记录为主，难以有效推理 |
| 中期 | 信息积累，部分善良开始交底 | 中 - 开始排除不可能的状态 |
| 后期（最后1-2轮） | 善良交底，信息趋于真实 | 高 - 可能性收敛，可做有效判断 |

## 推理的关键时机

真正有效的推理窗口是**游戏后期**——当善良玩家开始交底提供真实信息时：

- 善良玩家的信息可作为可靠基准
- 与这些信息产生矛盾的声称，其提供者更可能是邪恶 / 醉酒 / 中毒

这也是 BRD 中"从第 3 个白天开始触发推理"的理论依据。
